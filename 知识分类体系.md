# 知识分类体系

设计一个可扩展、易维护的知识分类体系，支撑个人知识库的持续积累。

## 分类原则

### MECE原则

所有分类应该**相互独立、完全穷尽**（Mutually Exclusive, Collectively Exhaustive）。

```
好的分类：编程 | 投资 | 写作 | 心理学（互不重叠）
坏的分类：技术 | 科技编程 | Python（重叠交叉）
```

### 三层结构

```
领域（Domain）→ 类别（Category）→ 主题（Topic）
   ↓               ↓                  ↓
  宽泛            中等               具体
```

### 动态调整

分类体系不是一成不变的，需要定期审视和调整：

- 新领域出现时 → 新增分类
- 子主题过多时 → 拆分
- 合并重复 → 归一化
- 长期无更新 → 归档

## 标签设计

### 标签命名规则

```
#type/xxx        # 笔记类型
#status/xxx      # 笔记状态
#领域/xxx        # 领域分类
#来源/xxx        # 信息来源
#重要程度/xxx    # 重要性
```

### 类型标签

```
#type/概念笔记      # 定义和解释一个概念
#type/读书笔记      # 书的内容总结
#type/项目笔记      # 项目进展记录
#type/日报          # 每日记录
#type/周报          # 每周总结
#type/会议记录      # 会议纪要
#type/流程文档      # 操作流程
#type/模板          # 可复用的模板
#type/人物档案      # 人物分析
```

### 状态标签

```
#status/进行中      # 还在完善
#status/初稿        # 首次完成
#status/已完成      # 内容完善
#status/需更新      # 内容过时
#status/待整理      # 需要整理
#status/归档        # 不再需要
```

### 来源标签

```
#来源/书籍          # 来自某本书
#来源/播客          # 来自播客
#来源/课程          # 来自在线课程
#来源/文章          # 来自文章
#来源/对话          # 来自沟通
#来源/思考          # 自己的想法
#来源/工具          # 使用工具时的笔记
```

### 重要程度

```
#重要程度/高        # 经常查阅的核心笔记
#重要程度/中        # 偶尔查阅
#重要程度/低        # 备查，很少看
```

### 组合使用

```
[[复利效应]] #type/概念笔记 #领域/投资 #来源/书籍 #status/已完成

[[2024周报W12]] #type/周报 #status/已完成

[[Python装饰器]] #type/概念笔记 #领域/编程 #重要程度/中
```

## 自动分类规则

### 关键词映射

```
关键词                                  → 分类
Python, JavaScript, Rust, 框架         → #领域/编程
股票, 基金, 定投, 复利, 资产配置       → #领域/投资
写作技巧, 标题, 结构, 故事             → #领域/写作
认知偏差, 决策, 思维模型               → #领域/心理学
```

### 标题模式匹配

```
标题包含「周报」「周复盘」             → #type/周报
标题包含「日记」「日志」               → #type/日记
标题包含「项目」「V1」「V2」           → #type/项目笔记
标题为「《》」开头的                   → #type/读书笔记
```

## 分类维护清单

**每周：**
- [ ] 检查新笔记的分类是否准确
- [ ] 调整误分类的笔记

**每月：**
- [ ] 审查标签使用情况，合并重复标签
- [ ] 检查是否有新领域需要添加
- [ ] 清理废弃标签

**每季度：**
- [ ] 审视分类体系是否仍然合理
- [ ] 拆分过于庞大的分类
- [ ] 归档不再使用的分类

**每年：**
- [ ] 全面回顾分类体系
- [ ] 输出分类体系调整记录
- [ ] 更新本文档
